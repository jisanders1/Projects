Short Paragraph

The ChocoAn Data center is strictly in charge of software actions the actors (provider, timer, manager, and operator) can perform. Other financial-related software actions will be performed through a third-party organization, Acme Accounting Services. The glossary includes all terms related to the elements and use cases in our UML case diagram. Our UML case diagram shows the relationship of the ChocoAn Data center’s use cases to its actors and the relationship of use cases to use cases.




Glossary

1.) Bills ChocAn - After a service is provided, the provider bills chocAn by verifying the member helped, date and service number.
2.) ChocAn ID Card - A plastic card that has the member's name and a nine-digit member number
3.) End of Week Report(sent to Member)- Contains member name, member number, member street address, city, state, zip code, and for each service: Date of service, provider name and service name
4.) End of Week Report(sent to Provider)- Contains provider name, number / id, street address, city, state, zip code and for each service: date of service, date and time that data was received by the computer, member name, member number, service code, fee to be paid. Also, Total number of consultations with members and total fee for week
5.) Interactive Mode- Configuration of the ChocAn data center that allows operators to add new members, delete members and update records. Also allows operators to do the same for provider. ChocAn data center is run in this mode during the day.
6.) Manager - Works with the ChocAn Data Center with main focus on the billing/fees side of the company
7.) Member number - nine-digit unique number to identify member. Used by member to validate membership in ChocAn.
8.) Operators - ChocAn workers that can add, delete and modify ChocAn memberships
9.) Provider - Assists customer with needs while operating with ChocAn devices to document and bill member interactions
10.) Provider directory- alphabetically ordered list of service names and corresponding service codes and fees; sent to the provider as an email attachment
11.) Provider number - unique identifying number assigned to each provider.
12.) Provider Terminal - A specially designed computer terminal (similar to POS systems) that each provider needs to use in order to serve members of ChocAn.
13.) Service Code- six digit code that corresponds to the type of service provided. For example, 598470 is the code associated with a dietitian.
14.) Summary Report(for manager of accounts payable) - The report lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week. Finally, the total number of providers who provided services, the total number of consultations, and the overall fee total are printed.
15.) Timer - At midnight on Friday, the timer activates the main accounting procedure
16.) EFT Reports - A record consisting of electronic funds transfer (EFT) data is then written to a disk; banking computers will later ensure that each provider’s bank account is credited with the appropriate amount.
17.) Operator number - unique identifying number assigned to each operator that is used to verify their role.
18.) Manager number - unique identifying number assigned to each manager that is used to verify their role.




User Case Descriptions

Title: Member Validation
Context: Before providing the health care service to the member, the provider needs to swipe the member’s card at the provider terminal, validating that the prospective customer is indeed a member of ChocAn.
Actors Member gives card to Provider; Provider swipes card
Main Success Scenario:
1.) The member gives the provider their nine-digit ChocAn member ID card.
2.) Provider swipes the card or enters the number at the provider terminal.
3.) The provider terminal communicates with the ChocAn Data Center which verifies the member number.
4.)The word “Validated” appears on the one-line display.
5.)The health care provider now provides the service to the member.
6.)Exit system
Extensions:
 4a. The number is not valid
  4a.1. The reason is displayed (invalid number or member suspended)



Title: Bill ChocAn
Context:After a member has completed a session with a provider, the provider swipes the members ID card. If verified, the provider documents the date of the appointment and accesses the provider directory to look up a six digit code indicating the service provided. The provider then enters the code and verifies the service recorded in the system. The system then displays the corresponding fee and displays it to the providers terminal. To verify the provider enters the date, basic member info, and the service provided.
Actors Provider
Main Success Scenario:
1.The terminal validates the member
2. After a service, the member hands the provider their card
3. The provider swipes or enters the members card
4. When VALIDATED appears, the provider enters the date the service was provided, given by MM-DD-YYYY.
5. The provider accesses the Provider Directory to find the corresponding six-digit service code to the service provided
6. The provider enters the code
  6.1. System displays up to a 20 character description of the code entered and gives the provider a chance to verify
  6.2 If an invalid code is entered, an error message is displayed
  6.3 Provider can add comments if needed
7. Fee associated for entered service is displayed to the provider’s terminal
8. To verify, the provider enters the current date and time, the date the service was provided, member name and number, service code and fee to be paid
9. At the end of the week, the provider totals the fees to verify how much money is to be paid by ChocAn
10. Exit system
Extentions: N/A

Title: Run Main Accounting Procedure
Context:At midnight on Friday, the main accounting procedure is run at the ChocAn Data Center.
Actors Timer
Main Success Scenario:
1.The main accounting procedure is automatically activated by a timer on Friday at midnight.
2.The main accounting procedure reads the week’s file of services provided.
3. Reports are generated for members if they were provided service during the week.
  3.1 Each report is sent to the respective member as an email attachment.
4.Reports are generated for providers with a list of services they provided
  4.1 Each report is sent to the respective provider as an email attachment.
5.A record consisting of electronic funds transfer (EFT) data is then written to a disk; banking computers will later ensure that each provider’s bank account is credited with the appropriate amount.
6.A summary report is given to the manager for accounts payable.
7.The main accounting procedure ends and the system exits.
8.Exit system

Title: Verify Operator
Context:When an operator’s terminal is booted up, they must verify that they are a valid operator before they can complete any of their actions.
Actors: Operator
Main Success Scenario:
1.) Assuming it is daytime, an operator terminal is booted up and prompts an operator to enter their operator number.
2.)Their number is a valid number.
3.)They are now able to choose the actions they wish to complete.
4.)Exit system
